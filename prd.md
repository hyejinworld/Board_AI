# 📄 제품 요구사항 문서 (PRD): 간단한 게시판 CRUD

## 1. 개요
본 프로젝트는 React(Vite), Node.js, EC2 상의 MySQL을 이용하여 게시판의 핵심 기능인 CRUD(생성, 읽기, 수정, 삭제)를 구현하는 것을 목표로 합니다. 클라이언트와 서버 간의 통신은 Axios를 이용한 REST API 기반의 JSON 데이터 교환으로 이루어집니다.

## 2. 요구사항

### 2.1 데이터 모델 (Board)
- **글번호 (id)**: 정수형 (INT), 자동 증가 (AUTO_INCREMENT), 기본 키 (Primary Key)
- **제목 (title)**: 문자열 (VARCHAR)
- **내용 (content)**: 텍스트 (TEXT)
- **작성자 (author)**: 문자열 (VARCHAR)
- **작성일 (created_at)**: 날짜 및 시간 (DATETIME), 기본값은 현재 시간 (CURRENT_TIMESTAMP)

### 2.2 핵심 기능 목록 (Frontend & Backend 연동)
1. **전체 검색 (게시글 목록 조회)**
   - 등록된 모든 게시글의 목록(글번호, 제목, 작성자, 작성일)을 최신순으로 조회하여 화면에 표시합니다.
   - API: `GET /boards`
2. **글번호 검색 (게시글 상세 조회)**
   - 특정 글번호를 클릭하여 해당 게시글의 상세 정보(제목, 내용, 작성자, 작성일 등)를 조회하고 화면에 표시합니다.
   - API: `GET /boards/:id`
3. **등록 (게시글 작성)**
   - 사용자가 제목, 내용, 작성자를 입력하여 새로운 게시글을 데이터베이스에 저장합니다.
   - API: `POST /boards`
4. **수정 (게시글 수정)**
   - 기존 게시글의 글번호를 기준으로 제목, 내용을 수정하여 데이터베이스에 업데이트합니다.
   - API: `PUT /boards/:id`
5. **삭제 (게시글 삭제)**
   - 특정 글번호를 선택하여 해당 게시글을 데이터베이스에서 영구 삭제합니다.
   - API: `DELETE /boards/:id`

## 3. 기술 스택 및 환경 설정
- **Frontend**: React (Vite 기반), HTTP Client (Axios)
- **Backend**: Node.js (Express 등)
- **Database**: MySQL (AWS EC2 환경에 설치됨)
- **API 형태**: RESTful API
- **데이터 포맷**: JSON

### 3.1 클라이언트 네트워크 및 환경 설정
- **Axios 인스턴스화**: `axios.create`를 활용하여 기본 Base URL 및 공통 헤더 등이 설정된 Axios 커스텀 객체를 생성해 API 호출에 재사용합니다.
- **환경 변수 관리**: 백엔드 API의 엔드포인트(URL) 등은 코드에 하드코딩하지 않고 `.env` 파일(Vite 환경의 경우 `VITE_API_BASE_URL` 등)을 사용하여 안전하고 유연하게 관리합니다.

## 4. UI/UX 요구사항
- **메인 페이지 (목록)**: 게시글 목록을 표 형태로 출력하며, '새 글 작성' 버튼을 제공합니다.
- **상세 페이지 (조회)**: 제목, 작성자, 내용, 작성일을 보여주며, '목록으로', '수정', '삭제' 버튼을 제공합니다.
- **작성 및 수정 폼**: 사용자가 데이터를 입력할 수 있는 필드(제목, 작성자, 내용)와 '저장/수정' 및 '취소' 버튼을 제공합니다.

## 5. 비기능적 요구사항
- **에러 처리**: API 통신 실패 및 데이터베이스 쿼리 에러 발생 시 사용자에게 명확한 피드백(에러 메시지 알림 등)을 제공해야 합니다.
- **개발 방식**: Vibe Coding 가이드에 따라 작게 만들고 빠르게 동작을 확인하며 명확한 구조를 유지해야 합니다.
